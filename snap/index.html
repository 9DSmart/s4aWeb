S4A goes lambda!

Welcome to the public pre-alpha release of Snap4Arduino!

Briefly, this is how Snap4Arduino differs from S4A:

The Good

  • Advanced programming concepts
  • You can Build Your Own Blocks™
  • Almost all Arduino boards supported (including Mega!)
  • Configurable pinouts

The Bad

  • No MacOSX version yet, but one will be available in a few days!
  • Buggy! Please report any bugs you find so we can go beta soon!
  • Not compatible with S4A
  • You can only program one board at a time (for the moment)

The Whatever

  • Web-based
  • Uses standard Firmata firmware
  • Javascript client, Pharo Smalltalk server

Nice! I want to try it!

There are currently versions for GNU/Linux and Microsoft Windows available. The project and all its components (including Snap!) are registered under public free software licenses (GPL and MIT), so you can download the sources and pretty much do whatever you want with them!

And how do I get it to work?

Snap4Arduino requires that you have StandardFirmata installed in your board.

To do so, follow these simple steps:

  1. If you haven't already, download and install the Arduino environment by following the instructions on http://arduino.cc/en/Main/Software.
  2. Open the Arduino IDE, go to File → Examples → Firmata → StandardFirmata
  3. Connect your board to a USB port in your computer
  4. In the Tools menu, select the board version and the serial port where the board is connected
  5. Go to File and click on Upload

Snap4Arduino will now be able to interact with your board.
